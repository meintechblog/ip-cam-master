## Summary

IP-Cam-Master v1.0 delivers one-click camera onboarding for non-ONVIF cameras (Mobotix, Loxone Intercom) into UniFi Protect. From a single web interface, users can discover cameras on the network, provision LXC containers on Proxmox with go2rtc transcoding and ONVIF servers, monitor camera health with real-time UniFi Protect integration, and install the entire system with a single command on a Proxmox host.

## Key Accomplishments

1. **Full Mobotix Pipeline** — MJPEG cameras transcoded to H.264 via VAAPI hardware acceleration in auto-provisioned LXC containers
2. **Loxone Intercom Support** — nginx auth-proxy pipeline handles authenticated streams, network discovery finds cameras automatically
3. **UniFi Protect Integration** — Real-time status from Protect API, SSH-based UDM log parsing for event detection, flapping alerts
4. **Dashboard** — Live camera monitoring with snapshot preview, container resource usage, event timeline, and Protect adoption status
5. **One-Line Installer** — Proxmox host script creates VM, provisions app, sets up API tokens and SSH keys automatically
6. **Access Control** — Session-based auth with setup wizard, login, and YOLO mode for homelab simplicity

## Decisions Made

- SvelteKit + SQLite (better-sqlite3) + Drizzle ORM — lightweight full-stack for homelab
- VAAPI hardware acceleration via /dev/dri passthrough to LXC containers
- Cloud-init based VM provisioning for the installer
- Dual UniFi integration: Protect API for status + SSH log parsing for events
- ONVIF cameras shown as "nativ nutzbar" — no workflow needed

## Known Issues / Tech Debt

- Phase 3 was implemented retroactively (Loxone + Discovery existed from prior work)
- 4 pre-existing test failures in proxmox.test.ts (SSH mock timeouts)
- UniFi Protect adoption requires manual click in Protect UI (no API endpoint for third-party camera adoption)
- DB_ENCRYPTION_KEY must be passed as env var in production (SvelteKit doesn't auto-load .env)
