   a. **Module-level cache** (place near the top of the file, after other top-level declarations):


   b. **Helper function** (place near other module-private helpers, or just above `listContainers`):
   ```ts
   function parseMacFromNet0(net0?: string): string | null {
     if (!net0) return null;
     const match = net0.match(/hwaddr=([0-9A-Fa-f:]{17})/i);
     return match ? match[1].toUpperCase() : null;
   }
   ```

   c. **`listContainers()`** (~lines 413-442): After obtaining `apiContainers` and before mapping to `ContainerInfo`:
   ```ts
   // Determine vmids missing from the MAC cache
   const uncachedVmids = apiContainers
     .map((c: any) => c.vmid as number)
     .filter((vmid: number) => !macCache.has(vmid));

   // Fetch configs in parallel; tolerate per-vmid failures
   if (uncachedVmids.length > 0) {
     await Promise.all(
       uncachedVmids.map(async (vmid: number) => {
         try {
           const cfg = await proxmox.nodes.$(node).lxc.$(vmid).config.$get() as any;
           const mac = parseMacFromNet0(cfg?.net0);
           if (mac) macCache.set(vmid, mac);
         } catch {
           // Skip — MAC will resolve to null for this vmid this round
         }
       })
     );
   }

   // Drop cache entries for vmids that no longer exist
   const liveVmids = new Set(apiContainers.map((c: any) => c.vmid as number));
   for (const vmid of macCache.keys()) {
     if (!liveVmids.has(vmid)) macCache.delete(vmid);
   }
   ```

   Then in the `result = apiContainers.map(...)` block, add `mac: macCache.get(c.vmid) ?? null` to the returned object (keep the `satisfies ContainerInfo`).

   d. **`getContainerStatus(vmid)`** (~lines 447-469): Ensure single-container queries also return `mac` consistently:
   ```ts
   // Populate MAC from cache; fall back to a one-off config fetch if missing
   if (!macCache.has(vmid)) {
     try {
       const cfg = await proxmox.nodes.$(node).lxc.$(vmid).config.$get() as any;
       const mac = parseMacFromNet0(cfg?.net0);
       if (mac) macCache.set(vmid, mac);
     } catch {
       // Ignore — mac stays null
     }
   }
   ```
   Then add `mac: macCache.get(vmid) ?? null` to the returned object.

   **Constraint**: The cache MUST NOT grow unbounded. The `listContainers()` diff step (removing entries whose vmid is no longer live) provides the only cleanup path — do not add a TTL or LRU. Per-container deletion via the existing `$delete` path is not required to touch the cache because `listContainers()` will sync on the next call.
