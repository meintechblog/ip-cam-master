<task type="checkpoint:human-action" gate="blocking">
  <name>Task 01: Provision bridge via wizard and verify ROADMAP success criteria 1-8</name>
  <files></files>

  <what>
    Open the app in browser at http://ip-cam-master.local:5173/settings (or the deployed URL).

    **Wizard Flow (SC-1, SC-7):**
    1. Navigate to the "Protect Hub" tab
    2. Click "Bridge einrichten" → should navigate to /settings/protect-hub/onboarding
    3. Step 1: Protect connection check should auto-run and show green checkmark (if creds are configured)
    4. Click "Weiter" → Step 2
    5. Read the LAN-trust info panel (SC-8: "Bridge-Endpunkte sind absichtlich nicht authentifiziert...")
    6. Click "Bridge bereitstellen" → wait for provisioning (10s with template, 3-5min without)
    7. On success: note the bridge IP shown

    **Container Verification (SC-2, SC-4, SC-5):**
    Run on the Proxmox host:
    ```bash
    # SC-2: Template reuse + bridge sizing
    pct config <vmid>
    # Verify: memory=1024, cores=2, /dev/dri mount, onboot=1

    # SC-4: YAML stamp
    pct exec <vmid> -- cat /etc/go2rtc/go2rtc.yaml
    # Verify: first line starts with "# managed by ip-cam-master, reconcile-id"

    # SC-5: Reboot test (OPTIONAL — only if convenient)
    # pve-manager reboot or: pct stop <vmid> && pct start <vmid>
    # Wait 60s, verify bridge IP unchanged and go2rtc responds
    ```

    **go2rtc Verification (SC-3):**
    From another LAN host:
    ```bash
    # API accessible
    curl http://<bridge-ip>:1984/api/streams
    # Should return JSON with "test" stream

    # Editor inaccessible
    curl http://<bridge-ip>:1984/editor.html
    # Should return 404 or empty (NOT the editor UI)

    # RTSP test stream
    ffprobe rtsp://<bridge-ip>:8554/test
    # Should show video stream metadata (libx264, 640x360, 10fps)
    ```

    **Lifecycle Controls (SC-6):**
    Back in browser, navigate to /settings → Protect Hub tab:
    1. Click "Stoppen" → status should change to "Gestoppt" within 10s
    2. Click "Starten" → status should change to "Läuft" within 10s
    3. Click "Neustarten" → brief "Gestoppt" then "Läuft" transition

    **Health Probe (SC-6):**
    Wait 5 minutes or check the DB:
    ```bash
    sqlite3 data/ipcm.db "SELECT last_health_check_at FROM protect_hub_bridges;"
    ```
    Should show a recent ISO timestamp.

    **Provision Failure / Retry (SC-7):**
    If you have a way to simulate failure (e.g., disconnect Proxmox storage briefly), verify:
    - Failed provision shows error in wizard with "Erneut versuchen" button
    - Re-clicking "Erneut versuchen" cleans up the failed row and re-provisions
    - No duplicate VMIDs created
    (This criterion can be accepted via code review if live failure simulation is impractical.)
  </what>

  <how-to-verify>
