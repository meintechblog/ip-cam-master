<locked_technical_decisions>
These are non-negotiable — decided in the phase context block, do NOT revisit:

1. **Detached systemd unit** — use `systemd-run --unit=ip-cam-master-update-<ts> --service-type=oneshot --collect --quiet /usr/local/bin/ip-cam-master-update.sh <preSha> <preSchemaHash>`. The `--service-type=oneshot --collect` form (NOT `--scope`) is chosen for reliability and queryability via `systemctl status ip-cam-master-update-<ts>.service`. Args are passed as argv array, never shell-interpolated.

2. **Script lives in repo, installed to system path** — `scripts/update.sh` is version-controlled. On server boot, `ensureUpdateScriptInstalled()` copies it to `/usr/local/bin/ip-cam-master-update.sh` + `chmod +x` IF the target is missing or older (mtime compare). This way the detached unit runs a script on the ROOT filesystem that does not disappear when the worktree changes during git pull.

3. **Progress via log file, not child_process pipe** — the in-memory pipe would be killed when the app restarts as part of the update. The script writes to `/tmp/ip-cam-master-update-<timestamp>.log` and the SSE endpoint tails the file via `fs.watch` (or fallback polling every 500 ms). The log file survives the restart; the EventSource reconnects automatically and the UI resumes reading the same file.

4. **Completion marker protocol** — the script writes (a) a final marker line `=== UPDATE_RESULT: success (<preSha> -> <postSha>) ===` or `=== UPDATE_RESULT: failed (<reason>, rolled back to <preSha>) ===` to the log, and (b) an exit code file `/tmp/ip-cam-master-update-<timestamp>.exitcode` containing `0`, `1`, or `2`. The SSE endpoint watches for the exitcode file to decide when to emit `event: done` and close the stream.

5. **Pre-update guards (server-side, before spawn)**:
   - `getCurrentVersion()` — reject with HTTP 409 `{ error: 'dirty_tree', dirtyFiles: string[] }` if `isDirty`. The dirty file list comes from `git status --porcelain` run via `execFile`.
   - `getCurrentVersion()` — reject with HTTP 400 `{ error: 'dev_mode' }` if `isDev`.
   - `getStoredUpdateStatus()` — reject with HTTP 400 `{ error: 'already_up_to_date' }` if `!hasUpdate`, UNLESS request body has `force: true` (allows retry after a failed run).
   - Capture `preSha = current.sha` (full) and `preSchemaHash = sha256(fs.readFile('src/lib/server/db/schema.ts'))` BEFORE spawning.

6. **Update script logic** (`scripts/update.sh`) — must match the template in the phase context block exactly:
   - `set -e`, trap on ERR
   - All stdout/stderr of each step teed to `$LOG`
   - Schema-hash compare AFTER git pull, BEFORE npm install; writes `WARNING: schema.ts changed` line and continues (non-blocking per UPDATE-08)
   - Rollback path on EVERY failure: `git reset --hard $PRE_SHA` then best-effort `npm install` + `npm run build` + `systemctl restart`
   - Writes exitcode file at the END of every branch
   - Wait 3 seconds after `systemctl restart` then `systemctl is-active` check

7. **Service layer (`src/lib/server/services/update-runner.ts`)**:
   ```typescript
   export function ensureUpdateScriptInstalled(): Promise<void>;
   export function spawnUpdateRun(preSha: string, preSchemaHash: string): Promise<{
     logPath: string;
     exitcodeFile: string;
     unitName: string;
     startedAt: string;
   }>;
   export async function* tailUpdateLog(
     logPath: string,
     exitcodeFile: string,
     signal: AbortSignal
   ): AsyncIterable<
     | { type: 'log'; line: string }
     | { type: 'done'; exitCode: number; result: 'success' | 'failed' | 'rolled_back' }
   >;
   export function getDirtyFiles(): Promise<string[]>;  // `git status --porcelain` parsed
   ```
   - `ensureUpdateScriptInstalled()` reads `scripts/update.sh` from the install dir (same candidate-dir logic as version.ts: `/opt/ip-cam-master` then `process.cwd()`), compares mtime to `/usr/local/bin/ip-cam-master-update.sh`, copies with `fs.copyFile` + `fs.chmod(0o755)` if missing/older. Silent no-op in dev mode.
   - `spawnUpdateRun()` uses `node:child_process.spawn` with `systemd-run` and argv array (NEVER shell interpolation). Detaches via `{ detached: true, stdio: 'ignore' }` and calls `child.unref()`.
   - `tailUpdateLog()` is an async generator. Strategy: open a read position at 0, read new bytes, split on `\n`, yield each line. Poll every 500 ms (simpler than fs.watch and works across filesystems). Between polls, check if exitcode file exists; when it does, drain remaining log content, read exit code, yield `done`, and return.
   - `getDirtyFiles()` execFiles `git status --porcelain` in the install dir and returns the trimmed lines.

9. **Endpoints**:
   - `POST /api/update/run` (`src/routes/api/update/run/+server.ts`):
     1. Check auth already handled by hooks.server.ts (route is NOT in isPublicPath)
     2. Parse body `{ force?: boolean }`
     3. Run guards: dev mode → 400, dirty tree → 409 with dirtyFiles, not-hasUpdate → 400 unless force
     4. Compute preSha (from version) + preSchemaHash (sha256 of schema.ts)
     5. Call `spawnUpdateRun` → get `{ logPath, exitcodeFile, unitName, startedAt }`
     6. `appendUpdateRun({ startedAt, finishedAt: null, preSha, postSha: null, result: 'running', logPath, unitName })`
     7. Return 202 `{ logPath, exitcodeFile, unitName, startedAt }`
   - `GET /api/update/run/stream?logPath=...&exitcodeFile=...` (`src/routes/api/update/run/stream/+server.ts`):
     1. **Path traversal protection**: validate both query params start with `/tmp/ip-cam-master-update-` and have no `..` — reject with 400 `{ error: 'invalid_path' }` otherwise
     2. Build a ReadableStream following the journal-stream pattern (start/cancel, request.signal abort handler, 15s heartbeat, TextEncoder)
     3. Inside start(): loop over `tailUpdateLog(logPath, exitcodeFile, signal)`; for each yielded item emit `event: log\ndata: <json-encoded {line}>` or `event: done\ndata: <json-encoded {exitCode, result}>`
     4. On `done`: parse the last marker line for postSha (regex: `/UPDATE_RESULT: success \(([0-9a-f]+) -> ([0-9a-f]+)\)/`), call `updateUpdateRun(unitName, { finishedAt, result, postSha })` (unitName derived from logPath basename), close stream
     5. On client abort: close the tail generator via AbortSignal, close controller
   - `GET /api/update/run/history` (`src/routes/api/update/run/history/+server.ts`): returns `readUpdateRuns(5)` as JSON

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/update/run | Authenticated user can trigger arbitrary git pull + build + restart |
| Browser → /api/update/run/stream | Authenticated user can request the SSE endpoint with attacker-controlled file paths |
| Node process → systemd-run → detached shell script | Node must pass pre-computed SHAs as argv, not via shell interpolation |
| scripts/update.sh → git remote (GitHub) | Trusts whatever git pull returns from origin/main |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | POST /api/update/run | mitigate | Auth-gated via hooks.server.ts (route not in isPublicPath). Verify in Task 3. |
| T-09-02 | Information Disclosure | GET /api/update/run/stream logPath param | mitigate | Regex validation: `^/tmp/ip-cam-master-update-\d+\.log$`, reject `..` or absolute paths outside /tmp/ip-cam-master-update-*. No fs access outside that glob. |
| T-09-03 | Elevation of Privilege | spawnUpdateRun argv | mitigate | All args (preSha, preSchemaHash) passed as argv array to child_process.spawn, NEVER shell-interpolated. No `{shell: true}`. |
| T-09-04 | Denial of Service | Multiple concurrent /api/update/run POSTs | mitigate | systemd-run with `--unit=ip-cam-master-update-<ts>` creates timestamped units; first one grabs the repo. Concurrent runs are naturally serialized by git (second git pull would fail fast). Accept that concurrent triggers produce at most one successful update. |
| T-09-05 | Repudiation | Who triggered the update? | accept | Single-user self-hosted tool. The update_run_history entries include timestamps; no multi-user audit needed. |
| T-09-06 | Spoofing | Git remote serving malicious code | transfer | Relies on HTTPS + GitHub account security. Out of scope for this app — same trust boundary as `git pull` from any self-hoster's workflow. |
| T-09-07 | Tampering | /tmp log files readable by other users on VM | accept | Single-tenant VM, root-owned ip-cam-master process. /tmp permissions are standard (1777). Log files are world-readable but only contain build output, no secrets. |
| T-09-08 | Denial of Service | Build failure leaves app broken | mitigate | scripts/update.sh rollback path re-runs `git reset --hard $PRE_SHA && npm install && npm run build && systemctl restart`. Task 5 tests this path. |
| T-09-09 | Information Disclosure | Schema change dump to user | accept | Only a warning text — hash change is not disclosed, just "schema.ts changed". No sensitive data in schema comparison. |
| T-09-10 | Tampering | scripts/update.sh on filesystem | mitigate | Script installed with chmod 0755 (not 0777). Only root can modify /usr/local/bin. ensureUpdateScriptInstalled only OVERWRITES with mtime-newer source, never appends. |
</threat_model>
