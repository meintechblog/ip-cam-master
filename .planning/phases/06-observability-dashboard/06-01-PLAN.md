<objective>
Add a Host Vitals surface to the main dashboard so the user can see disk usage, RAM usage, and `ip-cam-master.service` status (with uptime) at a glance — without SSH.

Per 06-RESEARCH.md Sections 5, 6, and 7: shell out to `df -B1 --output=size,used,avail,pcent /`, read `/proc/meminfo` directly (using `MemAvailable`, NOT `os.freemem()`), and parse `systemctl show ip-cam-master --property=...`. Zero new npm dependencies. No scheduler changes — piggyback on the existing 10s dashboard poll.

Purpose: Closes three of six observability requirements (HEALTH-01, HEALTH-02, HEALTH-03) and establishes the reusable `host-metrics` service layer that Phase 09 will reuse for post-update service verification.

Output: A new service module, a consolidated GET endpoint, a three-card Svelte component, and a wired-in dashboard.
</objective>

  <behavior>
    Tests (write FIRST, watch them fail, then implement):
    - getDiskUsage('/') parses `df -B1 --output=size,used,avail,pcent /` stdout with two lines (header + row), returns { totalBytes, usedBytes, availableBytes, percentUsed } as numbers (no % sign in the number).
      Mock: execFile returns "       1B-blocks         Used        Avail Use%\n     53687091200  12884901888  40802189312   24%\n"
      Expect: { totalBytes: 53687091200, usedBytes: 12884901888, availableBytes: 40802189312, percentUsed: 24 }
    - getMemoryUsage() reads /proc/meminfo, uses MemAvailable (NOT MemFree) to compute used = MemTotal - MemAvailable, returns percentUsed rounded to 1 decimal.
      Mock: fs.readFile returns "MemTotal:       16000000 kB\nMemFree:         1000000 kB\nMemAvailable:   10000000 kB\n"
      Expect: totalBytes = 16000000 * 1024, availableBytes = 10000000 * 1024, usedBytes = 6000000 * 1024, percentUsed = 37.5
    - getMemoryUsage() falls back to MemFree when MemAvailable is absent (old kernels). Graceful.
    - getServiceStatus() parses key=value output of `systemctl show ip-cam-master --property=ActiveState,SubState,ActiveEnterTimestamp,MainPID,Result --no-pager`. Computes uptimeSeconds from ActiveEnterTimestamp when ActiveState='active'.
      Mock: execFile returns "ActiveState=active\nSubState=running\nActiveEnterTimestamp=Thu 2026-04-10 14:32:11 CEST\nMainPID=1234\nResult=success\n"
      Expect: state='active', subState='running', mainPid=1234, result='success', uptimeSeconds > 0
    - getServiceStatus() returns uptimeSeconds=null when ActiveEnterTimestamp is empty OR when ActiveState !== 'active'.
      Mock: "ActiveState=inactive\nSubState=dead\nActiveEnterTimestamp=\nMainPID=0\nResult=success\n"
      Expect: state='inactive', mainPid=null, uptimeSeconds=null
    - getServiceStatus() handles unknown ActiveState gracefully (returns 'unknown' not a crash).
  </behavior>
  <action>
    Create src/lib/server/services/host-metrics.test.ts first with vitest. Use `vi.mock('node:child_process')` to stub `execFile` (which we promisify) and `vi.mock('node:fs/promises')` to stub `readFile`. Watch tests fail.

    Use `promisify(execFile)` from `node:util` — do NOT use `exec` (shell injection surface). Pass argv as an array.

    getDiskUsage: default path parameter is '/'. Parse the last non-empty line of stdout, split on whitespace, columns are [size, used, avail, pcent]. Strip the trailing % from pcent.

    getMemoryUsage: read /proc/meminfo as utf8, match each line against /^(\w+):\s+(\d+)\s+kB$/, build a record. Prefer kv.MemAvailable over kv.MemFree. Multiply kB by 1024 for bytes. Round percentUsed to 1 decimal: Math.round(x * 10) / 10.

    getServiceStatus: split stdout by \n, split each line on the FIRST '=' only (values may contain '='). Map ActiveState to the union type with 'unknown' fallback. Only compute uptimeSeconds when state==='active' AND ActiveEnterTimestamp is non-empty AND Date.parse returns a finite number.

    Run the tests again, confirm they pass.
