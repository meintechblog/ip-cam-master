<objective>
Ship version awareness and update-check for IP-Cam-Master so self-hosters see which commit they are running and learn — on demand or automatically — when `main` on `meintechblog/ip-cam-master` has moved forward.

Purpose: This is the read-only half of the v1.1 self-update story. It validates the git/GitHub data model in isolation (current SHA, latest SHA, last-checked-at, dirty state) before Phase 09 wires in the actual git-pull/build/restart runner. Zero execution risk, zero new dependencies — just `node:child_process.execFile` for git and built-in `fetch` for the GitHub API.

Output: A `version` service + `update-check` service with unit tests, two API endpoints, a Settings "Version" tab, and a header badge that lights up when a newer commit is available on main.
</objective>

Git commands to use (via node:child_process.execFile, NOT execSync):
```bash
git describe --tags --always --dirty --abbrev=7   # → 'v0.1.0-5-gabc1234' or 'abc1234' or 'abc1234-dirty'
git rev-parse HEAD                                 # → full 40-char SHA
```
Parse `--dirty` suffix to derive `isDirty: boolean`. Parse leading `v<tag>` to derive optional `tag`.

GitHub API contract (public, no auth needed):
```
GET https://api.github.com/repos/meintechblog/ip-cam-master/commits/main
Headers:
  Accept: application/vnd.github+json
  User-Agent: ip-cam-master-update-check
Response 200:
  { sha: string, commit: { committer: { date: string }, message: string }, ... }
Response 403 (rate limit):
  Headers: X-RateLimit-Remaining: 0, X-RateLimit-Reset: <unix-seconds>
Timeout: 10 seconds (AbortController)
```

Settings keys to use (store via settings service):
- `update_last_checked_at`  → ISO 8601 timestamp of last successful check
- `update_latest_sha`        → full 40-char SHA from GitHub API
- `update_latest_commit_date` → ISO 8601 from commit.committer.date
- `update_latest_commit_message` → commit.message (first line only, truncate to 200 chars)
- `update_last_error`         → 'rate_limited:<resetAt>' | 'network' | '' (empty when last check succeeded)

        hasUpdate: status.hasUpdate,
      });
    };
