Create `src/lib/server/services/protect.ts` implementing the UniFi Protect API client per D-01, D-04, D-05, D-06.

Use direct `fetch()` calls (not the unifi-protect npm package) per research recommendation.

**Session management (per D-04):**
- Module-level `session` variable: `{ cookies: string, csrfToken: string, expiresAt: number } | null`
- `login()`: POST to `https://{host}/api/auth/login` with `{ username, password }` from `getSettings('unifi_')`. Password decrypted via `decrypt()`. Extract cookies from `res.headers.getSetCookie().join('; ')` and CSRF token from `res.headers.get('x-csrf-token')`. Set expiresAt to `Date.now() + 8 * 60 * 1000` (8 min safety margin).
- `protectFetch(path)`: Check session validity, re-login if expired. GET `https://{host}/proxy/protect/api/${path}` with Cookie and X-CSRF-Token headers. On 401, re-login and retry once.
- Set `process.env.NODE_TLS_REJECT_UNAUTHORIZED = '0'` at module top for self-signed UDM cert.

**Camera fetching:**
- `getProtectCameras()`: Call `protectFetch('cameras')`, return typed `ProtectCamera[]`.

**Camera matching (per D-06):**
- `matchCamerasToProtect(ourCameras, protectCameras)`: Returns `Map<number, ProtectCameraMatch>`.
- Match pipeline cameras (vmid > 0) by `containerIp === protectCamera.host`.
- Match native ONVIF cameras (vmid === 0) by `cam.ip === protectCamera.host`.

**Aggregated status with 30s cache (per D-16):**
- Module-level cache: `let statusCache: { data: ProtectStatus, expiresAt: number } | null = null;`
- `getProtectStatus()`: If `statusCache` exists and `Date.now() < statusCache.expiresAt`, return cached data. Otherwise fetch Protect cameras, load our cameras from DB, match them, build `ProtectStatus` object with `connected: true`, `adoptedCount`, `connectedCount`, `totalProtectCameras`, and the matches Map. Cache result with `expiresAt = Date.now() + 30_000` (30s per D-16). This ensures the 10s frontend poll gets cached data and only actually hits the Protect API every 30s.
- If Protect API is unreachable (settings missing or login fails), return `{ connected: false, adoptedCount: 0, connectedCount: 0, totalProtectCameras: 0, cameras: new Map() }` — do NOT cache failures.

**ONVIF verification for adoption (per D-09):**
- `verifyOnvifServer(containerIp: string)`: Fetch `http://{containerIp}:8899` with 3s timeout. Return `{ running: boolean, reachable: boolean }`. Handle ECONNRESET as "running" (same pattern as cameras/status endpoint).

Export: `getProtectCameras`, `matchCamerasToProtect`, `getProtectStatus`, `verifyOnvifServer`, `protectFetch`.

**Create `src/lib/server/services/events.ts` (per D-12, D-13, D-14, D-15):**

Imports: `db` from client, `events` from schema, drizzle operators (`eq`, `and`, `gt`, `desc`, `sql`, `like`), types from `$lib/types`.

Functions:

1. `storeEvent(event: Omit<CameraEvent, 'id'>)`: Insert into events table. Return the inserted row.

2. `storeEvents(eventList: Omit<CameraEvent, 'id'>[])`: Batch insert. For each event, call `db.insert(events).values(...)`.

3. `getEvents(filters?: { cameraId?: number; severity?: EventSeverity; eventType?: EventType; since?: string; until?: string; limit?: number; offset?: number })`: SELECT from events with optional WHERE clauses. Default limit 50, default offset 0. Order by timestamp DESC (per D-26). Return `{ events: CameraEvent[], total: number }` where total is count without limit/offset.

4. `detectFlapping(cameraId: number): boolean` (per D-15): Query events where cameraId matches, eventType is `camera_disconnect`, and timestamp > `datetime('now', '-10 minutes')`. Return `count > 3`.

5. `getFlappingCameras(): number[]`: Query all camera IDs with >3 disconnects in last 10 minutes. Return array of camera IDs. Use SQL: `SELECT camera_id, COUNT(*) as cnt FROM events WHERE event_type = 'camera_disconnect' AND timestamp > datetime('now', '-10 minutes') GROUP BY camera_id HAVING cnt > 3`.

6. `cleanupOldEvents()` (30-day retention per research recommendation): `DELETE FROM events WHERE timestamp < datetime('now', '-30 days')`.

Export all 6 functions.

```typescript
const LOG_PATTERNS = {
  disconnect: /(\d{4}-\d{2}-\d{2}T[\d:.]+Z?)\s.*\[([^\]]+)\s*@\s*([\d.]+)\].*disconnect/i,
  reconnect: /(\d{4}-\d{2}-\d{2}T[\d:.]+Z?)\s.*\[([^\]]+)\s*@\s*([\d.]+)\].*reconnect/i,
  streamFailed: /(\d{4}-\d{2}-\d{2}T[\d:.]+Z?)\s.*stream.*(?:failed|error|timeout)/i,
  adopted: /(\d{4}-\d{2}-\d{2}T[\d:.]+Z?)\s.*(?:adopted|adoption)/i,
  aiportError: /(\d{4}-\d{2}-\d{2}T[\d:.]+Z?)\s.*aiport.*error/i,
};
```

Noise filters (per D-14):
```typescript
const NOISE_PATTERNS = [
  /ONVIF discovery/i,
  /health check/i,
  /go2rtc.*started/i,
  /periodic/i,
];
```

Functions:

1. `parseLogLines(stdout: string, since: Date)`: Split by newline, skip noise patterns, match against LOG_PATTERNS, extract timestamp/cameraName/IP, filter lines newer than `since`. Return array of `{ eventType, severity, message, cameraName, cameraIp, timestamp, source: 'ssh_logs' }`.

2. `scanUdmLogs()`: Connect to UDM via SSH using `getSettings('unifi_')` for host. SSH config: host from `unifi_host` setting, username `root`, privateKeyPath from `getSetting('udm_ssh_key_path')` falling back to `/opt/ip-cam-master/data/udm_key`. Run `tail -500 /srv/unifi-protect/logs/cameras.thirdParty.log` (primary log file). Parse with `parseLogLines` using `lastScanTimestamp` or 5-minutes-ago as since. Update `lastScanTimestamp` to now. Always call `ssh.dispose()` in finally block. Return parsed events array.

3. `fetchRawProtectLogs(lines: number = 100)`: On-demand raw log fetch for the Logs page "Protect Logs" tab (per D-27). SSH to UDM, run `tail -{lines} /srv/unifi-protect/logs/cameras.thirdParty.log`, return raw stdout string. Dispose SSH in finally.

Export: `scanUdmLogs`, `parseLogLines`, `fetchRawProtectLogs`.
