<objective>
Add runtime monitoring of `ipcam.tutk_server` to the existing MQTT subscriber so the
dashboard reflects cloud-mode toggles in real time (D-06). Without this, a user who
enables cloud mode on their A1 while it's adopted sees the stream go dead with no
explanation; after this, they see the same `A1_CLOUD_MODE_ACTIVE` badge as in preflight.

Purpose: make cloud-mode-active a first-class dashboard signal, not a silent failure.
Output:
- `BambuConnectionError` type extended with `'A1_CLOUD_MODE_ACTIVE'`
- Message handler at `bambu-mqtt.ts:104-117` extended with the field watch
- `lastError = null` unconditional reset at line 106 becomes conditional
- Vitest: 4 new tests (edge-trigger enable, edge-trigger disable, conditional-reset, no-op on undefined)
</objective>

           if (typeof gcodeState === 'string' && gcodeState) {

           // NEW (Phase 18 / D-06 / BAMBU-A1-06): TUTK cloud-mode runtime watch
           const tutkServer = msg?.print?.ipcam?.tutk_server;
           if (typeof tutkServer === 'string') {
               if (tutkServer === 'enable' && sub.lastError !== 'A1_CLOUD_MODE_ACTIVE') {
                   sub.lastError = 'A1_CLOUD_MODE_ACTIVE';
