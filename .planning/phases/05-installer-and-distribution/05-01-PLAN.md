    **2. Create auth service** at `src/lib/server/services/auth.ts` (per D-26):
    - `hashPassword(password: string): string` -- use Node.js built-in `crypto.scryptSync` (consistent with existing crypto.ts pattern) with random salt, store as `salt:hash` hex string
    - `verifyPassword(password: string, stored: string): boolean` -- split stored value, re-derive, compare with `timingSafeEqual`
    - `createSession(): string` -- `randomBytes(32).toString('hex')` returns 64-char token
    - `validateSession(token: string): boolean` -- check token exists in in-memory session map (Map<string, {username, expiresAt}>), sessions expire after 24 hours
    - `destroySession(token: string): void` -- remove from map
    - `isSetupComplete(): boolean` -- check if a user row exists in users table
    - `getUser(): {username: string} | null` -- get the single user record
    - `createUser(username: string, password: string): void` -- insert or update user row (upsert, since only 1 user per D-27)
    - `deleteUser(): void` -- delete the user row (for removing login per D-25)
    - Session storage: in-memory Map (sessions lost on restart = acceptable for homelab, forces re-login after update/restart)

    **3. Create auth test** at `src/lib/server/services/auth.test.ts`:
    Test hashPassword, verifyPassword, createSession, session lifecycle (create, validate, expire, destroy).

    **4. Create hooks.server.ts** (per D-24):
    - Import `validateSession`, `isSetupComplete` from auth service
    - For every request: check if setup is complete (user exists)
      - If NO user exists: redirect all non-setup routes to `/setup` (allow `/setup` and static assets)
      - If user exists: check cookie `session` for valid session token
        - Valid session: continue
        - No/invalid session: redirect to `/login` (allow `/login` route)
    - YOLO mode (per D-23): if no user row exists AND a YOLO flag setting `auth_yolo` is set to `true` in settings table, skip all auth checks (allow everything)
    - Export as `handle` function using SvelteKit's `Handle` type

    **1. Create systemd service file** at `ip-cam-master.service` (per D-13 through D-17):
    ```ini
    [Unit]
    Description=IP-Cam-Master
    After=network.target

    [Service]
    Type=simple
    WorkingDirectory=/opt/ip-cam-master
    ExecStart=/usr/bin/node build/index.js
    Restart=on-failure
    RestartSec=5
    Environment=NODE_ENV=production
    Environment=PORT=80
    EnvironmentFile=/opt/ip-cam-master/.env
    StandardOutput=journal
    StandardError=journal
    SyslogIdentifier=ip-cam-master

    [Install]
    WantedBy=multi-user.target
    ```

    **2. Create install.sh** (per D-01 through D-08, D-09, D-18 through D-21):

    Script structure:
    ```
    #!/usr/bin/env bash
    set -euo pipefail
    ```

    **Header:** Print banner "IP-Cam-Master Installer". Check running as root (exit if not).

    **Detect mode** (per D-02): Check if `/opt/ip-cam-master` exists.
    - EXISTS = update mode
    - NOT EXISTS = fresh install mode

    **Fresh install flow** (per D-03):
    1. Install system dependencies: `apt-get update && apt-get install -y curl git ffmpeg`
    2. Install Node.js 22 LTS via NodeSource: `curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && apt-get install -y nodejs` (per INST-02, D-03)
    3. Clone repo: `git clone https://github.com/meintechblog/ip-cam-master.git /opt/ip-cam-master` (per INST-01)
    4. `cd /opt/ip-cam-master`
    5. `npm install --omit=dev` (production only)
    6. Create data directory: `mkdir -p data`
    7. Generate .env (per D-20): Create `/opt/ip-cam-master/.env` with random `DB_ENCRYPTION_KEY=$(openssl rand -hex 32)`. Only create if .env doesn't exist.
    8. Run build: `npm run build`
    9. Run DB migration: `npx drizzle-kit push` (per D-18)
    10. Copy systemd service: `cp ip-cam-master.service /etc/systemd/system/`
    11. `systemctl daemon-reload && systemctl enable ip-cam-master && systemctl start ip-cam-master`
    12. **SSH key setup** (per D-05 through D-08):
        - Ask: "Proxmox Host IP eingeben (oder Enter zum Ueberspringen):"
        - If provided: generate ed25519 key at `/root/.ssh/ip-cam-master` if not exists (`ssh-keygen -t ed25519 -f /root/.ssh/ip-cam-master -N ""`)
        - Run `ssh-copy-id -i /root/.ssh/ip-cam-master.pub root@$PROXMOX_IP` (user enters password once)
        - Print: "SSH-Key wurde eingerichtet. Kein Passwort mehr noetig."
    13. Print success: "IP-Cam-Master laeuft auf http://$(hostname -I | awk '{print $1}')"

    **Update flow** (per D-04, D-21):
    1. `cd /opt/ip-cam-master`
    2. `systemctl stop ip-cam-master`
    3. `git pull` (per INST-03)
    4. `npm install --omit=dev`
    5. `npm run build`
    6. `npx drizzle-kit push` (per D-18, auto-migration)
    7. Copy service file (may have changed): `cp ip-cam-master.service /etc/systemd/system/`
    8. `systemctl daemon-reload && systemctl start ip-cam-master`
    9. Print: "Update erfolgreich. IP-Cam-Master wurde neu gestartet."

    **Error handling:** Use `set -euo pipefail`. Add trap for cleanup messages on failure. Each major step prints what it's doing ("Installiere Node.js 22...", "Baue Anwendung...", etc.).
