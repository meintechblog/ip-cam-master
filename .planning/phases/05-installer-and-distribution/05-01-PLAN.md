    **2. Create auth.ts service (per D-26: scryptSync hashing, per D-23: YOLO mode, per D-24: session cookies):**
    - `hashPassword(password: string): string` -- uses `crypto.scryptSync` with random 16-byte salt, returns `salt:hash` in hex. Use `scryptSync(password, salt, 64)` with `{ N: 16384, r: 8, p: 1 }` options.
    - `verifyPassword(password: string, storedHash: string): boolean` -- splits stored hash on ":", re-derives with same salt, uses `timingSafeEqual` for comparison.
    - Session management: module-level `const sessions = new Map<string, { userId: number; username: string; expiresAt: Date }>()`. Session TTL = 24 hours.
    - `createSession(username: string): string` -- generates `randomBytes(32).toString('hex')`, stores in Map with 24h expiry, returns session ID.
    - `validateSession(sessionId: string): SessionData | null` -- checks Map, validates expiry, deletes if expired.
    - `deleteSession(sessionId: string): void` -- removes from Map.
    - `isYoloMode(): boolean` -- reads `auth_yolo` setting from DB via `getSetting('auth_yolo')`, returns `value === 'true'`.
    - `getUser(): { id: number; username: string } | null` -- queries users table, returns first row or null.
    - `createUser(username: string, password: string): void` -- hashes password, upserts into users table (single user per D-27: delete existing first, then insert).
    - `deleteUser(): void` -- deletes all rows from users table.
    Import db from `$lib/server/db/client`, users from schema, getSetting from settings.

    **4. Update hooks.server.ts:**
    Replace the current content (just `startScheduler()`) with:
    ```typescript
    import { startScheduler } from '$lib/server/services/scheduler';
    import { validateSession, isYoloMode, getUser } from '$lib/server/services/auth';
    import type { Handle } from '@sveltejs/kit';

    startScheduler();

    const PUBLIC_PATHS = ['/setup', '/login', '/api/auth', '/api/settings'];

    export const handle: Handle = async ({ event, resolve }) => {
      const path = event.url.pathname;

      // Allow public paths
      if (PUBLIC_PATHS.some(p => path.startsWith(p))) {
        return resolve(event);
      }

      // YOLO mode skips all auth (per D-23)
      if (isYoloMode()) {
        return resolve(event);
      }

      // No user exists -> redirect to setup (per D-22)
      const user = getUser();
      if (!user) {
        return new Response(null, { status: 303, headers: { location: '/setup' } });
      }

      // Check session cookie
      const sessionId = event.cookies.get('session');
      if (!sessionId) {
        return new Response(null, { status: 303, headers: { location: '/login' } });
      }

      const session = validateSession(sessionId);
      if (!session) {
        event.cookies.delete('session', { path: '/' });
        return new Response(null, { status: 303, headers: { location: '/login' } });
      }

      // Attach user to locals for downstream use
      event.locals.user = session;
      return resolve(event);
    };
    ```

    NOTE on /api/settings in PUBLIC_PATHS: This is required because install.sh POSTs Proxmox credentials to /api/settings on fresh install before any user exists. Without this, the "no user -> redirect to /setup" logic would return a 303 redirect instead of accepting the curl POST. This is safe because the app runs on a local network only, and the settings API requires knowing the DB encryption key context to store sensitive values.
