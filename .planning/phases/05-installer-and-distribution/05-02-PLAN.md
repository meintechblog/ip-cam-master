  <action>
    Complete rewrite of install.sh. The script runs on the **Proxmox host** (not inside a VM). Structure follows the architecture from RESEARCH.md.

    **Constants at top:**
    ```bash
    VM_NAME="ip-cam-master"
    VM_TAG="ip-cam-master"
    REPO_URL="https://github.com/meintechblog/ip-cam-master.git"
    APP_DIR="/opt/ip-cam-master"
    PVE_USER="ipcm@pve"
    PVE_TOKEN_NAME="ipcm"
    PVE_ROLE="IPCamMaster"
    IMG_URL="https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-genericcloud-amd64.qcow2"
    IMG_CACHE="/var/lib/vz/template/iso/debian-12-genericcloud-amd64.qcow2"
    ```

    **Helper functions:**
    - `banner()` -- show IP-Cam-Master Installer header
    - `step()` -- prefix output with ">>"
    - `error_exit()` -- print error and exit 1
    - `confirm()` -- read y/N confirmation, exit on N

    **Pre-checks:**
    - Must run as root
    - Must be on a Proxmox host: check `command -v pvesh` exists, otherwise error "Dieses Skript muss auf einem Proxmox-Host ausgefuehrt werden."
    - Ensure `jq` is installed: `apt-get install -y -qq jq 2>/dev/null || true`

    **detect_existing_vm() function (per D-02):**
    Use `pvesh get /cluster/resources --type vm --output-format json` piped to `jq` to find a VM with tag "ip-cam-master". Return VMID or empty string.
    ```bash
    detect_existing_vm() {
      pvesh get /cluster/resources --type vm --output-format json 2>/dev/null \
        | jq -r '.[] | select(.type=="qemu" and .tags!=null and (.tags | contains("ip-cam-master"))) | .vmid' \
        | head -1
    }
    ```

    **get_vm_ip(vmid) function:**
    Use `qm agent $vmid network-get-interfaces` with jq to extract the first non-loopback IPv4 address. Retry up to 30 times with 5s sleep. Error if no IP found.

    **wait_for_ssh(ip) function:**
    Try `ssh -o StrictHostKeyChecking=no -o ConnectTimeout=3 root@$ip echo ok` up to 30 times with 5s sleep.

    **Mode detection (per D-05):**
    ```bash
    EXISTING_VMID=$(detect_existing_vm)
    if [ -n "$EXISTING_VMID" ]; then
      echo "Bestehende Installation gefunden (VM $EXISTING_VMID)."
      echo ""
      echo "  [U] Update — Neueste Version installieren"
      echo "  [R] Remove — VM und Konfiguration entfernen"
      echo "  [C] Cancel — Abbrechen"
      echo ""
      read -rp "Auswahl [U/R/C]: " CHOICE
      case "${CHOICE,,}" in
        u) MODE="update" ;;
        r) MODE="remove" ;;
        *) echo "Abgebrochen."; exit 0 ;;
      esac
    else
      MODE="install"
    fi
    ```

    **fresh_install() function (per D-03, D-07, D-08, D-09, D-10, D-11, D-13):**

    1. Security warning (per D-07, D-09):
       ```
       echo "WARNUNG: Dieses Skript wird:"
       echo "  - Eine VM auf diesem Proxmox-Host erstellen"
       echo "  - Einen API-Token fuer die VM erstellen"
       echo "  - Der VM SSH-Zugriff auf diesen Host geben"
       echo ""
       confirm "Moechtest du fortfahren? (y/N)"
       ```

    2. Detect storage (per Research pitfall 1):
       ```bash
       NODE=$(hostname)
       STORAGE=$(pvesh get /nodes/$NODE/storage --output-format json \
         | jq -r '[.[] | select(.content | contains("images"))] | .[0].storage')
       ```
       Error if empty: "Kein passender Storage gefunden."

    3. Download Debian 12 cloud image (cached):
       ```bash
       if [ ! -f "$IMG_CACHE" ]; then
         step "Lade Debian 12 Cloud-Image herunter..."
         wget -q --show-progress "$IMG_URL" -O "$IMG_CACHE"
       fi
       ```

    4. Create VM (per D-10, D-11):
       ```bash
       VMID=$(pvesh get /cluster/nextid)
       step "Erstelle VM $VMID ($VM_NAME)..."
       qm create "$VMID" \
         --name "$VM_NAME" \
         --tags "$VM_TAG" \
         --memory 2048 --cores 2 \
         --net0 virtio,bridge=vmbr0 \
         --scsihw virtio-scsi-pci \
         --ostype l26 \
         --agent enabled=1
       ```

    5. Import disk + cloud-init:
       ```bash
       qm set "$VMID" --scsi0 "$STORAGE:0,import-from=$IMG_CACHE"
       qm set "$VMID" --ide2 "$STORAGE:cloudinit"
       qm set "$VMID" --boot order=scsi0
       qm set "$VMID" --serial0 socket --vga serial0
       qm disk resize "$VMID" scsi0 10G
       ```

    6. Cloud-init config:
       ```bash
       # Generate installer SSH key if needed
       INSTALLER_KEY="/root/.ssh/ipcm_installer"
       if [ ! -f "$INSTALLER_KEY" ]; then
         ssh-keygen -t ed25519 -f "$INSTALLER_KEY" -N "" -q
       fi
       qm set "$VMID" --ciuser root
       qm set "$VMID" --sshkeys "${INSTALLER_KEY}.pub"
       qm set "$VMID" --ipconfig0 ip=dhcp
       qm set "$VMID" --nameserver "$(grep nameserver /etc/resolv.conf | head -1 | awk '{print $2}')"
       ```

    7. Start VM + wait for IP + wait for SSH:
       ```bash
       qm start "$VMID"
       step "Warte auf VM-Start..."
       VM_IP=$(get_vm_ip "$VMID")
       step "VM-IP: $VM_IP"
       step "Warte auf SSH-Zugriff..."
       wait_for_ssh "$VM_IP"
       ```

    8. Provision app inside VM via SSH (per D-12, D-17, D-18, D-19, D-20, D-21):
       SSH into VM and run:
       - `apt-get update && apt-get install -y curl git ffmpeg qemu-guest-agent`
       - Install Node.js 22 LTS via NodeSource
       - `git clone $REPO_URL $APP_DIR && cd $APP_DIR`
       - `npm install --omit=dev`
       - `mkdir -p data`
       - Generate .env with random DB_ENCRYPTION_KEY (per D-19)
       - `npm run build`
       - `npx drizzle-kit push` (per D-21)
       - Copy ip-cam-master.service to /etc/systemd/system/, enable service (don't start yet)
       - Generate VM-to-host SSH key (per D-14): `ssh-keygen -t ed25519 -f /root/.ssh/ip-cam-master -N ""`
       - Print the public key back to stdout

    9. Add VM's public key to host's authorized_keys (per D-15):
       Capture the public key from the SSH output, append to `/root/.ssh/authorized_keys` on the host.

    10. Create API token (per D-08):
        ```bash
        # Create user
        pveum user add "$PVE_USER" --comment "IP-Cam-Master service account" 2>/dev/null || true
        # Create role with minimal permissions
        pveum role add "$PVE_ROLE" --privs \
          "VM.Allocate,VM.Audit,VM.Config.Disk,VM.Config.CPU,VM.Config.Memory,VM.Config.Network,VM.Config.Options,VM.PowerMgmt,VM.Console,Datastore.AllocateSpace,Datastore.Audit,SDN.Use" \
          2>/dev/null || pveum role modify "$PVE_ROLE" --privs \
          "VM.Allocate,VM.Audit,VM.Config.Disk,VM.Config.CPU,VM.Config.Memory,VM.Config.Network,VM.Config.Options,VM.PowerMgmt,VM.Console,Datastore.AllocateSpace,Datastore.Audit,SDN.Use"
        # Delete existing token if any
        pveum user token remove "$PVE_USER" "$PVE_TOKEN_NAME" 2>/dev/null || true
        # Create token
        TOKEN_OUTPUT=$(pveum user token add "$PVE_USER" "$PVE_TOKEN_NAME" --privsep 1 --output-format json)
        TOKEN_SECRET=$(echo "$TOKEN_OUTPUT" | jq -r '.value // .data.value // empty')
        if [ -z "$TOKEN_SECRET" ]; then
          error_exit "API-Token konnte nicht erstellt werden."
        fi
        TOKEN_ID="${PVE_USER}!${PVE_TOKEN_NAME}"
        # Set ACLs
        pveum acl modify / --user "$PVE_USER" --role "$PVE_ROLE"
        pveum acl modify "/nodes/$NODE" --tokens "$TOKEN_ID" --role "$PVE_ROLE"
        pveum acl modify /storage --tokens "$TOKEN_ID" --role "$PVE_ROLE"
        ```

    11. Inject Proxmox settings into VM (token + host IP):
        SSH into VM, use `node -e` with better-sqlite3 to insert settings into the DB:
        - `proxmox_host` = host IP (from `hostname -I | awk '{print $1}'`)
        - `proxmox_token_id` = TOKEN_ID
        - `proxmox_token_secret` = TOKEN_SECRET
        - `proxmox_ssh_host` = host IP
        - `proxmox_ssh_user` = root
        - `proxmox_ssh_key_path` = /root/.ssh/ip-cam-master
        Then start the service: `systemctl start ip-cam-master`

    12. Print success message with VM IP:
        ```
        echo "======================================"
        echo "  Installation erfolgreich!"
        echo "  IP-Cam-Master: http://${VM_IP}"
        echo "  VM-ID: ${VMID}"
        echo "======================================"
        ```

    **update_vm() function (per D-04, D-06):**
    1. Get VM IP via `get_vm_ip`
    2. SSH into VM:
       - `cd /opt/ip-cam-master`
       - `systemctl stop ip-cam-master`
       - `git pull`
       - `npm install --omit=dev`
       - `npm run build`
       - `npx drizzle-kit push` (per D-21)
       - `cp ip-cam-master.service /etc/systemd/system/`
       - `systemctl daemon-reload && systemctl start ip-cam-master`
    3. Check if API token works (per D-06): try `pvesh get /version --output-format json` with the token. If it fails, re-create token and inject.
    4. Print update success message.

    **remove_vm() function (per D-05):**
    1. Confirm: "VM $EXISTING_VMID und alle Daten werden geloescht. Fortfahren? (y/N)"
    2. Stop and destroy VM: `qm stop $EXISTING_VMID 2>/dev/null || true; qm destroy $EXISTING_VMID --purge`
    3. Remove API user/token: `pveum user token remove "$PVE_USER" "$PVE_TOKEN_NAME" 2>/dev/null || true; pveum user delete "$PVE_USER" 2>/dev/null || true`
    4. Remove role: `pveum role delete "$PVE_ROLE" 2>/dev/null || true`
    5. Clean up installer SSH key from host if desired
    6. Print removal success.

    **Main flow:**
    ```bash
    case "$MODE" in
      install) fresh_install ;;
      update) update_vm ;;
      remove) remove_vm ;;
    esac
    ```

    **Error handling:**
    - `set -euo pipefail` at top
    - `trap` for cleanup: if fresh_install fails partway, attempt to destroy the partially created VM
    - Each major step has a `step "..."` message for progress visibility
